# Web UI ツアー — 実機 Simulator を操作し、あらゆる証跡を収集する

これは **実機 Simulator で Bajutsu を評価する iOS 開発者**向けのデモです。ローカルの Web UI
（[`bajutsu serve`](../../docs/ja/cli.md)）を使って Simulator を起動し、同梱のサンプルアプリに対して
シナリオを実行し、各実行が捕捉する証跡 — スクリーンショット、動画、デバイスログ、通信、ビジュアル
リグレッションの差分、システムアラート処理 — をブラウザで辿ります。すべて**合否の判定に AI を介在
させません**。

まずセットアップ不要で同じ物語を見たい場合（Mac/Simulator 不要）は、
[60秒ツアー](../tour/README.ja.md) を実行してください: `uv run python demos/tour/tour.py`。

## 前提条件

- Xcode と起動できる Simulator のある macOS（`open -a Simulator`）。
- idb バックエンド — Web UI が必要時に自動インストールしますが、事前に入れることもできます:
  `brew install facebook/fb/idb-companion && uv sync --extra idb`。
- システムアラート部分のみ: `ANTHROPIC_API_KEY`（環境変数か gitignore された `.env`）。それ以外は
  完全に決定論的で、キーは不要です。

アプリを先にビルドする必要は**ありません** — Web UI が、`.app` が無い初回に設定された `build` コマンドを
実行し、選んだデバイスへインストールします。

## 起動

```bash
make -C demos webui                  # http://127.0.0.1:8765  （変更は PORT=8766）
```

（`make -C demos webui` は `make -C demos/features serve` に委譲します — どちらでも動きます。）

これはリポジトリ直下の `make serve` をラップし、[`demo.config.yaml`](demo.config.yaml) をバインドします。
この設定が Web UI を `sample` アプリと、その [`app/scenarios/`](app/scenarios) にあるシナリオに向けます。
表示された URL を開いてください。

> Web UI は必ず `make serve`（素の `bajutsu serve` ではなく）経由で起動してください: ラッパーが idb
> クライアント＋ companion を必要時にインストールするので、クローン直後でも
> `no available actuator among ['idb']` で失敗しません。

## ツアー

UI には2つのタブがあります: **Replay**（シナリオを決定論的に実行し、レポートを見る）と **Record**
（自然言語の目標からエージェントでシナリオを著作する）。以下のツアーは、特記なき限り Replay です。

### 1. シナリオを実行してレポートを読む

**Replay → Run** で: `sample` アプリを選び、一覧から Simulator を選び（停止中なら自動で起動されます。
「booted」はすでに起動中のものを指します）、`smoke.yaml` を選んで **Run**。ライブログがデバイスを起動し、
初回はアプリをビルド/インストールし、実行する様子が見えます。完了すると、対話的な**レポート**が埋め込み
表示されます。

レポートの **Result** パネルを開くと: 各ステップに、操作対象の**スクリーンショット**と**アクセシビリティ
ツリー**が、一致した要素をハイライトした状態で表示されます。**Expectations** テーブルには、合否を決めた
機械的なアサーションが並びます。これが中心的な約束です — ステップごとに監査できる決定論的な判定です。

### 2. 通信の記録（観測、そしてモック）

[`network.yaml`](app/scenarios/network.yaml) を実行すると、レポートに **Network** タブが増え、アプリが
行った各 HTTP(S) のやり取り — メソッド、URL、ステータス、所要時間、ヘッダ、ボディ — が表示されます。
これは BajutsuKit が TLS の*後*に捕捉します（プロキシも CA も不要）。このシナリオは**リクエストに対する
アサーション**（`request:` の期待）も持つので、通信そのものが決定論的なチェックの一部になります。

次に [`network_mock.yaml`](app/scenarios/network_mock.yaml) を実行: 同じ画面ですが、`mocks:` ルールが
実エンドポイントの代わりに定型のレスポンス（例: ステータス `418`）を返します。Network タブはそのやり取りを
**mocked** と示します — ライブ依存のない決定論的な通信です。

### 3. ビジュアルリグレッション（VRT）— そして UI からベースラインを承認

[`visual.yaml`](app/scenarios/visual.yaml) を実行します。初回は**ベースラインが無い**ため `visual` の期待は
意図的に失敗し、レポートには捕捉したスクリーンショットと **「Approve as baseline」**ボタンが表示されます。
クリックすると、そのスクリーンショットが baselines フォルダへ昇格します。再実行すると、今度はそのベース
ラインと**ピクセル比較**して合格します。後で画面がずれた場合は、レポートが差分割合つきの**並列差分**
（swipe / onion / blend モード）を表示します。`threshold` がアンチエイリアスの揺れを許容し、`exclude` の
矩形が変動領域（時計、ステータスバー）をマスクします。

> CLI での同じ承認手順: `make -C demos/features vrt` のあと `vrt-approve`。

### 4. システムアラート — SpringBoard のプロンプトを突破する

idb のアクセシビリティ照会は前面アプリしか見えないため、iOS のシステムプロンプト（「パスワードを保存し
ますか？」、権限ダイアログ）は実行を静かにブロックします。**Dismiss alerts** を有効（既定。
`ANTHROPIC_API_KEY` が必要）にして [`permission.yaml`](app/scenarios/permission.yaml) を実行すると、
プロンプトがステップをブロックしたときにガードがスクリーンショットを撮り、Claude vision にどこが閉じる
ボタンかを尋ね、タップして、ステップを再試行します。レポートの手順テーブルには、アラートのラベルを伴う
**「system alert dismissed」**の行が出ます。ここが `run` で AI が補助する唯一の箇所であり、しかも
*デバイスを操作する*だけで、判定には決して関与しません。シナリオは `dismissAlerts: { instruction:
"tap Allow" }` で、閉じる代わりに許可することもできます。

### 5. 動画とデバイスログ

[`evidence.yaml`](app/scenarios/evidence.yaml) を実行します。その `capturePolicy` がシナリオ全体の区間を
記録するので、レポートに **Video** タブ（シークバー付きの画面録画）と **Log** タブ（検索できるデバイス
ログ）が、`os_signpost` 区間のアプリ **Trace** とともに加わります。「X のたびに記録」をルールとして一度
書けば、AI なしで毎回再現されます — 設定としての証跡です。

### 6. 自然言語から新しいシナリオを著作する（Record タブ）

**Record** タブに切り替え: 目標（「カウンターを2回増やして 2 になることを確認」）を入力し、アプリと
エージェントを選んで **Record**。エージェントがライブ画面を読み、一手ずつ提案します（ストリーミング:
推論 → アクション）。実行されたステップは決定論的なシナリオとして書き出され、編集して **Save** し、
Replay タブから再生できます。これが Tier 1（AI 著作）から Tier 2（決定論的再生）への受け渡しです —
アーキテクチャ全体が一つのウィンドウに収まっています。

## このデモが示すもの

- **信頼でき、監査できる一つの判定** — 合否は機械的なアサーションだけで決まり、それを裏付ける
  スクリーンショット／要素ツリー／通信／動画が各ステップに添付されます。
- **証跡の幅広さ** — UI の状態、通信（観測とモック）、ビジュアルリグレッション、システムアラート、動画、
  ログを、その場限りのスクリプトではなく再利用可能なルールで捕捉します。
- **AI は役立つところに、判定には決して** — 著作（Record）とアラート処理は*操作*を補助し、ゲートは決定論
  のままです。
