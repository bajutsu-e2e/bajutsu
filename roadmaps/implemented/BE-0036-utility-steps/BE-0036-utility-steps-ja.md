## はじめに

`http` ステップは、ランナーから HTTP リクエストを発行し（テストデータの準備、webhook の起動、
後続ステップが必要とする値の取得のために）、レスポンス本文を `vars.*` 名前空間へ保存します。この
項目が当初一緒に対象としていた OTP／2FA とメールの側方チャネルステップは、別の提案へ分割しました
（[参考](#参考)を参照）。

## 動機

run はしばしば、UI 自身では供給できない値に依存します。バックエンドからのトークン、API で仕込む
フィクスチャ、run が起こさねばならない webhook などです。リクエストを発行してその結果を run に
取り込む手段がなければ、こうしたケースは手作業のセットアップ（再現できません）に頼るか、省かれて
しまいます。`http` はその欠落を埋めます。シナリオが値を取得し、後続のステップへランタイム変数
として決定的に渡せるようにします。

## 詳細設計

`http` は（アプリではなく）ランナープロセスからリクエストを発行し、レスポンス本文をランタイム
変数として保存します。後続で `${vars.*}` 補間に使えます。

```yaml
- http: { method: POST, url: "${secrets.API}/login", body: "...", status: 200, saveBody: token }
- type: { text: "${vars.token}", into: { id: auth.token } }
```

フィールドと契約：

- **`method`**（既定 `GET`）、**`url`**（`http`／`https` のみ）、任意の **`headers`** と
  **`body`**。
- **`status`**：指定すると、レスポンスのステータスがそれと照合され、食い違えばステップは失敗
  します。
- **`saveBody`**：レスポンス本文の**全体**を `vars.<saveBody>` として保存します。（本文全体では
  なく JSON パスや正規表現で単一フィールドを抽出する機能は、自然な続きとして今後の作業に挙げて
  おり、まだ未実装です。）
- これは*取得ユーティリティ*であり、アプリ自身のトラフィックを観測するアプリ内の `request`
  アサーション／`mocks` とは別物です。同じ PR では、デバイスをリセットする補助ステップ
  `clearKeychain` と `clearClipboard` も追加しました。

prime directive の保持：

- **run パス上に LLM を置かない。** `http` は決定的なリクエストで、その結果は機械的なルール
  （ステータスチェック、本文の取得）で取り込まれます。合否判定は引き続き機械チェック可能な
  アサーションのみから得られ、run／CI ゲートは AI 非依存のままです。
- **決定性。** リクエストは即時です。失敗したリクエストやステータスの食い違いは、黙った誤った値
  ではなく、きれいなステップ失敗として表面化します。
- **アプリ非依存。** ステップとその配線はアプリをまたいで同一で、異なるのはエンドポイントと
  シークレット（config／`${secrets.*}`）だけです。シークレット（API のベース URL、トークン）は
  `${secrets.X}` として参照し証跡では自動マスクされるので、シナリオはトークンを保持し値は保持
  しません。
- **codegen。** `http` はアプリではなく bajutsu ランナーで動き、XCUITest 等価物がないため、codegen
  はラベル付き `// TODO` を出力します
  （[BE-0026](../../proposals/BE-0026-shrink-unsupported-syntax/BE-0026-shrink-unsupported-syntax-ja.md) に
  従います）。

## 検討した代替案

- **任意のコマンドを実行して値を生成する汎用 `shell`／`exec` ステップ。** 柔軟性は最大ですが、
  シナリオを非可搬かつレビュー不能にし（何でも実行できる）、テストを CI ホストへ結びつけます。
  少数の目的特化で監査可能なステップを優先し、却下します。
- **`mocks`／`request` アサーションで値を観測する。** これらはアプリ*自身*のトラフィックをスタブ
  または観測するもので、テスト自身が帯域外で値を取得しなければならない場合には役立ちません。
  置き換えではなく補完として維持します。
