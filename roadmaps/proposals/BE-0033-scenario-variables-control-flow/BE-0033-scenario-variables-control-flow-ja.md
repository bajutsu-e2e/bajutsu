# BE-0033 — シナリオ変数 + 軽い制御フロー

* 提案: [BE-0033](BE-0033-scenario-variables-control-flow-ja.md)
* 状態: **可決・実装中**
* トラック: [可決済み](../../README-ja.md#可決済み)
* トピック: 競合調査（MagicPod / Autify）由来の候補
* 由来: MagicPod

## はじめに

`${...}` 補間プリミティブ（`interp.py`、params/row/secrets を共通処理）は実装済みです。残りは、UI 値を取得して後続のステップで `vars.*` として再利用する機能と、決定性を損なわない範囲での条件分岐・ループです。

## 動機

フローによっては、テストが事前に知り得ない値に依存します。チェックアウト後に表示される注文番号、生成されたユーザー名、次の画面が反映し返す件数などです。現状シナリオは、書かれたときのリテラルにしかアサートできないため、こうしたケースはそもそも表現できません。関連する欠落として、形状のバリエーションもあります。バナーがあるときだけ別の経路を取るフローや、ステップを上限つきで繰り返すケースです。ランタイムの値を取得して後で再利用すること、そして厳格な範囲内で分岐・ループすることが必要です。ただしシナリオを汎用プログラムに変えてはなりません。それは決定性とレビュー可能性を損なうからです。

## 詳細設計

共通の補間プリミティブは実装済みです。`${...}` トークンは `interp.interpolate` / `interp.find_tokens`（`bajutsu/interp.py`）によって名前空間をまたいで一様に解決され、`params.*` と `row.*` はロード時に、`secrets.*` / `vars.*` はアクション時に置換されます。`bindings` マップに存在するトークンは置換され、キーが存在しないものはそのまま残るため、各層は自分の名前空間だけを置換します。

**ランタイム変数（`vars.*`）—— 実装済み。** ステップの `extract` 修飾子は、ステップ実行後に UI 要素のプロパティをランタイム変数に取得します。各エントリは変数名を付け、`sel`（セレクタ）と任意の `prop`（`value` | `label` | `identifier`、デフォルト `value`）を与えます。`_run_extract`（`bajutsu/orchestrator/`）が `resolve_unique` でセレクタを解決し、結果を run を通じて持ち回る可変の束縛マップに `vars.<name>` として格納します。セレクタが一意に解決できないかプロパティが `None` の場合、ステップは失敗します。後続のステップとシナリオレベルの `expect` は、そのマップから `${vars.<name>}` を補間します（`_interp_step` / `_interp_asserts`）。取得は決定的なプロパティ読み取りと置換にすぎないため、run/CI の判定は機械のみのままで、LLM は関与しません。

**範囲を限定した制御フロー —— 残り。** 提案の粒度で、まだ設計が残っている部分はこうです。ライブ UI に対する機械検証可能な述語（要素が存在する、`vars.*` の値が何かと等しい、など）が成り立つときだけステップを実行する条件分岐と、ブロックを静的に上限の決まった回数だけ繰り返すループ（固定回数、あるいは「条件成立まで、ただし N で打ち切り」）です。指針となる制約はプライムディレクティブそのものです。あらゆる述語は決定的に機械評価されること（LLM の判断は決して使わない）、あらゆるループは run が必ず停止するようハードな上限を持つこと、スキップまたは短絡された分岐はレポートが実際に実行されたものを正確に反映できるよう透明に報告されること。分岐がロード時のデータだけに依存する場合は展開時の解決を優先し、真にランタイム依存の分岐だけを run ループに置きます。

## 検討した代替案

**シナリオに汎用のスクリプト／式言語を露出する。** 不採用。任意の式と上限のないループは、シナリオを発散させたり、ランナーが上限を定められないロジックに合否を依存させたりして、決定性優先を壊し、レビューを難しくします。代わりに、狭く範囲の限定されたプリミティブ集合だけを許します。

**取得と分岐をツール外部のドライバスクリプトに押し出す。** 不採用。テストロジックを共有のハブであるシナリオ YAML の外へ移し、実行されるフローがファイルやレポートで完全には見えなくなります。`extract`/`vars.*` と範囲限定の制御フローをシナリオ内に保てば、フロー全体がレビュー可能で再現可能なままです。

**LLM にランタイムで分岐を決めさせる。** 即座に不採用。モデルを Tier-2 の run/CI ゲートに入れることになり、「AI は決して判定しない」に反します。制御フローの述語はすべて機械検証可能でなければなりません。
