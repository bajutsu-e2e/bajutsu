# BajutsuSample アプリ 機能ガイド

このドキュメントは、サンプルアプリ（BajutsuSample）が「どんな画面で、ユーザーが何をできるか」を
おおまかに解説するものです。自然言語のテストシナリオを起こすための土台として読めるよう、
実装の詳細（識別子やコードの仕組み）ではなく、画面の役割とユーザー操作の流れに絞っています。

> 識別子・検証プリミティブまで含む詳細仕様は [`../README.md`](../README.md) を参照してください。

## アプリ全体像

iOS 向けの小さな SwiftUI アプリです。起動すると **オンボーディング → ログイン → ホーム** と進み、
ホーム以降は画面下部のタブバーで 10 個の画面を切り替えます。各タブは UI 部品やアプリ挙動の
1 ジャンルを担当し、操作の結果が画面上に表示されます。

## 起動から本編までの流れ

| 画面 | 内容 | ユーザー操作 |
|---|---|---|
| オンボーディング | 「Welcome」歓迎画面 | 「Get Started」を押して次へ |
| ログイン | メールとパスワードの入力 | 両方を入力して「Log in」。空欄のままだと送信できず、不正ならエラー表示 |
| ホーム（本編） | タブバーのある本体画面 | 以降は各タブを行き来して操作 |

※ ログインを済ませた状態やオンボーディングを飛ばした状態でも起動できる（テスト用の起動オプションあり）。
特定のタブや設定画面を最初から開いて起動することもできます。

## タブごとの画面と機能

### 1. Home（ホーム）
アプリのトップ。シンプルな操作を一通り含みます。
- カウンターの数字を「+」ボタンで増やす
- 「Load」ボタンで読み込みを開始し、スピナー表示のあと「Loaded」になる
- 検索欄に文字を入れてリストを絞り込む
- 画面右上の「Settings」から設定シートを開く

**設定シート**：「Normalize」のオン/オフ切り替え、変更後に出る「再インデックスが必要」バナー、
「Reindex」ボタンで再インデックス（しばらくして「完了」表示）、「Close」で閉じる。

### 2. Components（コンポーネント）
基本的なジェスチャと確認ダイアログの寄せ集め。
- 「Hold me」を長押しすると隠れていたテキストが現れる
- 「Remove」を押すと削除確認アラートが出て、「Delete」/「Cancel」を選ぶ
- 「Swipe me」エリアを上下左右にスワイプすると、その方向が結果表示される

### 3. Controls（コントロール）
標準的な入力コントロールのギャラリー。各操作の現在値が下に表示されます。
- トグル（オン/オフ）
- ステッパー（数値の増減）
- スライダー（0〜10 の段階）
- セグメント選択（One / Two / Three から 1 つ）
- メニュー（Apple / Banana を選ぶ）
- 押せるボタン（タップ数を数える）と、常に押せない無効ボタン

### 4. Text（テキスト入力）
文字入力のバリエーションと入力検証。
- 基本のテキスト欄（入力値と文字数を表示、「Clear」で消去）
- メールアドレス用の入力欄
- 複数行のテキストエディタ
- 必須入力欄（3 文字以上で有効）：短いとエラー表示、有効になるまで「Submit」は押せない

### 5. Lists（リストとナビゲーション）
一覧表示まわりの操作一式。
- 検索欄で行を絞り込み（一致なしのときは空状態を表示）
- 行を左スワイプして削除
- 「Edit」ボタンで編集モードに入り、並べ替え・削除
- 引っ張って更新（pull-to-refresh）で元の一覧に戻り「Refreshed」表示
- 行をタップすると詳細画面へ遷移
- 表示中の件数を常に表示

### 6. Gestures（ジェスチャ）
マルチタッチ系のジェスチャ。結果は方向や回数で表示。
- ダブルタップ（回数を数える）
- ピンチ（広げる/縮める＝ in / out）
- 回転（時計回り/反時計回り＝ cw / ccw）

### 7. Present（モーダル表示）
さまざまなモーダルの出し方。
- 「Open Sheet」：高さを変えられるシート（中の「Close」で閉じる）
- 「Open Full-Screen」：全画面カバー
- 「Open Dialog」：アクションシート（Archive / Delete / Cancel）、選んだ結果を表示
- 「Show Toast」：一定時間で自動的に消えるトースト表示

### 8. Async（非同期処理）
時間のかかる処理や待ち合わせ。
- 「Start」で進捗バーが 0% から 100% まで進み「Complete」表示
- 「Load (fails)」はわざと失敗し、「Retry」で再試行すると成功する
- 検索欄は入力が落ち着いてから（デバウンス後に）結果を反映
- 「Load more」で表示件数を増やす（ページング）

### 9. System（システム連携）
OS 機能との連携。アプリ外に出る操作を含む。
- 通知の許可リクエスト（システムの許可ダイアログが出る）
- コピー＆ペースト（クリップボード経由で貼り付け結果を表示）
- 共有（システムの共有シートを開く）

### 10. Network（ネットワーク）
実際の HTTP 通信を行うデモ。
- 「Fetch」で GET リクエストを送り、ステータスを表示
- 「GET query」「POST body」「DELETE」で各種リクエストパターンを送信
- 直近の通信内容（メソッド・ステータス・所要時間・URL など）を画面下に表示

## シナリオを書くときのヒント

- 多くの画面で「操作 → 結果が画面に表示される」構造になっているので、
  「○○を操作したら△△が表示される」という形のシナリオに落とし込みやすいです。
- 入力検証（ログイン、Text の必須欄）やボタンの有効/無効は、
  「条件を満たすまで送信できない」系のシナリオに向いています。
- Async タブは待ち時間を含むため、「結果が出るまで待つ」系のシナリオに向いています。
- System / Network はアプリ外（OS のダイアログや通信）に関わるため、
  シナリオでは前提（許可ダイアログの扱いやネットワーク環境）を意識してください。
